Resampling techniques

This is and addendum for last weekâ€™s questions.

Why resampling? Fitting machine learning models is not the end of the story. We want to evaluate our models to check if they should be applied in practice.

Evaluating on the datasets they were fitted leads to overfitting. we then split into to dataset into training and test data, where the test dataset is unknown for our model. Therefore, more reliable techniques would be desirable. However if there us no natural splitting criteria, the results are highly dependent on the split.

Bootstrap

If one have a dateset, we can take datapoints randomly, drawing with replacement from the original dataset. This construct two datasets. The idea is to be able to draw one datapoint multiple times.

If we split each bootstrap sample in training and test data, we obtain test errors for each one of them.

Cross-validation

In bootstrapping is not guaranteed that each datapoint is used equally often. This may be a problem in medical applications because no Â«patientsÂ» should be more important than another one. For this scenario, cross-validation uses each data point equally often.

The idea of \(K\)-fold validation is to split the data into \(K\) folds. Them the model is fitted on \(K-1\) folds, and the missing \(K\)th fold is predicted.

This is repeated \(K\) times, i.e. each fold is exactly once the testing dataset. One obtains \(K\) predictions errors that can be averaged.

How to choose \(K\)? The larger \(K\), the large is the variance of the estimated error. On the other hand, the larger \(K\), the better the cross-validation error approximates the true prediction error. In practice, often \(K=5%\) or \(K=10\) are chose.

There are common mistakes. Imagine a multiple procedure, e.g. variable selection and subsequent model building. The wrong way of doing cross-validation here would be to first do the variable selection and cross-validate only the model building. Cross-validation must always be performed on the entire procedure, otherwise, the prediction error is underestimated because the variables were chosen based on the entire dataset.

Some popular terms: leave-one-out cross-validation is \(n\)-fold cross-validation. The jackknife is another resampling technique. Here, \(d<n\) values are deleted from the dataset and model fitting and evaluation is performed on the resampling \(n-d\) values by splitting them in training and testing data.

The R-package caret is a great way to automatically perform cross-validation. In Python, the scikitlearn library can be used.

In random forest, we obtain the OOB value. Usually we do not have an OOB sample so we canâ€™t use the OOB error for hyperparameter tuning. Then, we can apply cross-validation and choose the hyperparameter set with the lowest cross-validation error. For this purpose, the R-package caret can be used.

Adaptive two-stage desings

Two-armed phase III clinical trial with experimental (E) and control (C) groups. We assume the outcome to be normally distributed, the common variance is sigma squared. Test on mena difference \(\mu_E - \mu_C\) the null hypothesis is \(H_0 = \mu_E - \mu_C \leq 0\).

Sample size in a one-stage design

Gial: power of \(1-\beta\) on effect size \(\theta_A > 0\). This can be reached by choosing the sample size \(n\) as follows: $$ n=2*\frac{(\Phi^{-1}(1-\beta) + \Phi^{-1}(1-\alpha/2))^2}{(\theta_A/\sigma)^2} $$

Two-stage design

The idea is to include an interim analysis and recalculate the samole size. Then, one has to choose:

  • First-stage smaple size \(n_1\).
  • Second-stage sample size \(n_2\).
  • Decision rule for the first stage.
  • Decision rule for the second stage.

Early stopping

Early stopping is an attractive feature of an adaptive design. It allows to terminate the trial at the interim analysis if the results are already very clear. There are two possible scenarios: (1) Early termination due to enormously well results (early stopping for efficacy), (2) enormously bad results (early stopping for futility). In terms of the first stage test statistic \(Z_1\), with \(n_1\) subjects:

  • If \(Z_1 < c_f\) we stop early for futility.
  • If \(Z_1 > c_e\) we stop early for efficacy.
  • If \(c_f \leq Z_1 \leq c_e\) enter the second stage.

How we choose \(c_f\) and \(c_e\)? We use the typo one error rate. $$ P_{\theta=0}[Z_1 > c_e] + P_{\theta=0}[c_f \leq Z_1 \leq c_e] * P_{\theta=0}[Z_2 > c_2] = \alpha $$ There are several rules to obtain this. The rule by Pocock, the rule of Oâ€™Brien, etc. The important thing is that, in the end, whenever we perform the second-stage trial with \(n_2\) subjects, if \(Z_2 > c_2\) we reject \(H_0\).

Now, \(n_2\) and \(c_2\) can be chosen in dependence on \(Z_1\). Therefore, these two parameters are functions of \(Z_1\).
